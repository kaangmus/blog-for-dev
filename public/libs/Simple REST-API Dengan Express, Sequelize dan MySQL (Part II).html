Dalam tutorial sebelumnya kita telah berhasil membuat operasi retrieve pada REST-API yang kita buat. Dalam bagian ini kita akan melanjutkan tutorial sebelumnya dengan menambahkan operasi create, update dan delete pada REST-API yang kita buat.

Pertama mari kita install beberapa package baru berikut ke dalam project kita:

  • body-parser: middleware yang digunakan untuk mem-parsing request body
  • express-validator: digunakan untuk validasi input form
  • multer: middleware untuk menangani mulipart/form data (kita gunakan untuk upload gambar)

Silahkan klik link untuk melihat info lebih lanjut dari package di atas

Buka Terminal/Command Prompt dan pastikan Anda sedang berada di root directory project. Ketika perintah berikut (dan enter) untuk menambahkan package di atas ke dalam project kita.

npm install --save body-parser express-validator multer

Setelah proses installasi package selesai, kita akan melakukan beberapa langkah berikut:

  1. Me-load semua package yang diperlukan
  2. Menggunakan body-parser sebagai middleware
  3. Set directory public menjadi static assets agar bisa bisa diakses melalui URL.
  4. Membuat multer storage engine untuk menangani file upload. Ini kita gunakan untuk upload gambar buku.

Mari kita tambahkan code berikut pada file index.js.

  • Pada baris 1–6 kita me-load semua package yang kita butuhkan ke dalam index.js
  • Pada baris 9–10 kita gunakan untuk menggunakan middleware body-parser untuk menangani request body
  • Baris 13 kita gunakan untuk mengeset directory public yang telah kita buat sebelumnya menjadi static assets. Dalam directory public kita telah membuat directory dengan nama img yang kita gunakan untuk menyimpan gambar buku yang disimpan dalam field image.
  • Pada baris 14–24 kita membuat multer storage engine untuk menangani proses file upoad yang dalam hal ini adalah gambar buku. Gambar buku yang terupload melalui API akan disimpan pada directory img di dalam directory public.
  • Pada baris 26 Kita definisikan multer storage engine dengan nama upload

Selanjutnya mari kita buat endpoint untuk menambahkan data ke dalam tabel book yaitu POST /book .

Anda bisa perhatikan code di atas. Dalam code di atas kita membuat sebuah route yang menerima tipe request POST ke path/book dalam server kita. Pada baris 3, kita menambahkan upload sebagai middleware untuk menangani multipart/form-datakarena dalam route ini, kita akan melakukan upload file gambar buku. Selanjutnya setiap parameter yang dikirimkan dari client, kita akan lakukan validasi dengan express-validator. Route POST /book akan menerima parameter data sebagai berikut:

  • isbn
  • name
  • year
  • author
  • description
  • image (optional)

Selanjutnya apabila terdapat error pada proses validasi, pada baris 29–31 kita akan langsung mengembalikan response sesuai dengan error yang digenerate oleh express-validator.

Jika tidak terdapat validation error, proses akan kita lanjutkan dengan menyimpan data sesuai dengan parameter yang dikirimkan oleh client. Untuk dapat mengakses request body, kita dapat menggunakan req.body.{nama parameter} . Kita menggunakan method create() untuk menyimpan data baru ke dalam tabel book. Jika sukses, pada baris 41–45 kita akan mengembalikan response dengan field data berisikan data buku yang baru saja kita masukan.

Mari kita coba endpoint POST /book yang baru saja kita buat dengan menggunakan Postman dengan mengirimkan parameter di atas.

Oke, sekarang kita telah berhasil membuat endpoint/route untuk operasi create. Mari kita tambahkan endpoint/route untuk operasi update untuk dapat melakukan update data buku.

Route PUT /book dengan parameter yang sama dengan parameter yang kita gunakan pada route POST /book . Untuk mengupdate data buku, kita berpatokan pada parameter isbn . Parameter isbn kita asumsikan sebagai unique. Data buku akan diupdate apabila value parameter isbn yang dikirimkan oleh client valid (tersedia dalam tabel book).

Mari kita tambahkan endpoint/route PUT/book pada index.js dengan menyisipkan code berikut:

  • Pada baris 2–25 kita menambahkan middleware untuk menangani file upload dan melakukan validasi. Proses ini sama dengan proses yang kita lakukan sebelumnya pada route POST /book
  • Apabila terdapat error pada proses validation pada parameter request PUT, pada baris 29–31 kita akan langsung memberikan response kepada client yang berisikan pesan eror yang digenerate oleh express-validator.
  • Jika tidak terdapat error. Pada baris 32–39 kita mendefinisikan data buku yang akan kita update, kemudian pada baris 40 kita akan melakukan update data buku berdasarkan ISBN dengan value dari parameter isbn
  • Selanjutnya, setelah proses update berhasil, kita akan mengembalikan JSON response dengan field status, messagedan data . Field data berisikan data book yang barusan kita update.

Silahkan jalankan ulang server Anda dengan node index.js dan lakukan test request PUT /book dengan menggunakan Postman. Berikut merupakan contoh request PUT /book untuk mengupdate data buku pada tabel book melalalui REST-API yang kita buat.

Oke tahap terakhir, mari kita buat endpoint/route untuk menghapus data buku dalam tabel book melalui REST-API yang kita buat. Untuk menghapus data buku dalam tabel book, kita akan melakukannya dengan menggunakan ISBN sebagai key. Oleh karena itu, kita akan menambahkan isbnsebagai route parameter.

Tambahkan code di bawa ini ke dalam file index.js.

  • Pada baris pertama, Anda bisa melihat kita mendefinisikan sebuah route baru dengan tipe request DELETE, di mana route ini menerima isbn sebagai route parameter.
  • Pada baris 3–13 kita akan melakukan pengecekan apakah parameter isbn valid dan memang ada dalam tabel book. Jika tidak valid, kita akan langsung memberikan response error.
  • Jika parameter isbn valid, kita akan melakukan penghapusan data buku dari tabel book. Untuk menghapus data dari tabel book, kita bisa menggunakan method destroy() . Pada baris 15 kita melakukan penghapusan data dari tabel book berdasarkan parameter isbn yang dapat kita akses dengan menggunakan req.params.isbn.
  • Jika tidak ada error, kita akan langsung memberikan JSON response.

Oke mari kita test endpoint/route DELETE /book/:isbn yang baru saja kita buat dengan menggunakan Postman.

Selamat! Anda telah berhasil membuat REST-API dengan operasi CRUD dengan Express, Sequelize dan MySQL. Sekian tutorial kali ini dan terima kasih telah membaca. Stay tuned untuk tutorial berikutnya.
