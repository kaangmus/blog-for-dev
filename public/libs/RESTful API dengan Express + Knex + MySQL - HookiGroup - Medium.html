RESTful API dengan Express + Knex + MySQL

Buda Suyasa
Feb 14 Â· 6 min read

Jika kamu adalah backend developer NodeJS, kamu pasti akan sering berhubungan dengan basis data. Baik basis data SQL seperti MySQL, SQLite, Postgree ataupun basis data NoSQL seperti MongoDB.

Pada artikel ini saya ingin membagikan bagaimana cara membuat sebuah RESTful API sederhana dengan Express dan Knex dengan menggunakan basis data MySQL.

Kamu mungkin tidak asing lagi dengan Express. Express adalah framework NodeJS untuk membangun website, dilengkapi dengan berbagai fitur standar seperti routing, middleware, dan templating. Sedangkan Knex adalah SQL query builder yang bisa kita gunakan untuk menuliskan SQL query agar terlihat lebih rapi dan mudah dibaca.

Knex adalah query builder yang sangat saya sukai dan saya sudah gunakan di beberapa project dalam perusahaan. So far, saya saya sangat puas dengan performa dan kemudahannya. Knex memiliki dokumentasi yang lengkap dan mudah dipelajari, sehingga sangat cocok bagi pemula yang baru belajar membuat web dengan menggunakan NodeJS.

Saat artikel ini ditulis, Knex telah mendukung berbagai jenis basis data diantaranya: Postgres, MSSQL, MySQL, MariaDB, SQLite3, Oracle, dan Amazon Redshift.

Oke mari kita langsung saja praktekan cara membuat RESTful API sederhana menggunakan Express, Knex dan MySQL.

Saya akan membuat sebuah RESTful API dengan nama tokobuku. Dalam proses pembuatan RESTful API ini, untuk mempermudah pemahaman pembaca, saya hanya akan menggunakan 1 file saja yaitu index.js.

Pertama saya akan membuat sebuah Project NodeJS baru dengan nama tokobuku. Melalui jendela console (command prompt), saya akan menggunakannpm init untuk membuat sebuah project baru.

npm init akan meminta saya untuk memasukan informasi mengenai project yang saya buat. Saya akan memberikan nama tokobuku, sedikit deskripsi dan menentukan entry pointnya bernama index.js.

Kemudian, langkah selanjutnya saya akan menginstall modul express, knex dan mysql dengan perintah berikut:

npm install --save express knex mysql

Untuk memudahkan proses coding, saya akan menggunakan nodemon untuk menjalankan server. Mari kita install nodemon terlebih dahulu dengan cara:

npm install -g nodemon

Kemudian saya akan membuat sebuah file baru dengan nama index.js pada root directory project, kemudian mengcopy-paste code di bawah untuk memastikan express sudah terinstall dengan benar.

Saya akan menjalankan server index.js di atas dengan menggunakan nodemon. Dari root directory project kita tinggal ketikan perintah:

nodemon index.js

Oke server sudah berjalan di http://localhost:3000. Oke mari kita check dengan menggunakan Web Browser.

Voilla!!! Server sudah berjalan. Next, saya akan membuat sebuah database baru dengan menggunakan MySQL. Database ini akan saya beri nama tokobuku. Agar sederhana cukup dengan 1 tabel yaitu tabel buku.

Struktur data dari tabel buku bisa dilihat pada tabel di bawah.

Dalam RESTful API yang saya buat ini, saya akan membuat 4 buah endpoint untuk melakukan manipulasi data pada tabel buku (CRUD). Detail dari setiap endpoint bisa dilihat pada tabel di bawah ini.

Pertama saya akan melakukan inisiasi instance module knex ke dalam index.js. Knex sebenarnya adalah sebuah function yang menerima argument object konfigurasi database. Berikut konfigurasi yang saya gunakan.

Selanjutnya mari kita buat endpoint pertama yaitu GET /buku untuk mengambil semua data buku dari table buku.

Pada code diatas saya membuat endpoint (route) /buku yang bisa diakses dengan HTTP method GET. Pada baris ke-3 saya menggunakan knex untuk mengambil seluruh data penulis dari table buku dengan functionknex('buku') . Saat eksekusi, function ini akan melakukan query:

select * from buku

Return value dari function di atas adalah object (array), yang saya tampung dalam variabel buku. Terakhir kita akan memberikan response dengan res.json() dengan variabel buku sebagai argumentnya.

Mari kita coba test endpoint di atas dengan menggunakan Postman.

Selanjutnya saya akan menambahkan endpoint POST /buku, untuk menambahkan data buku baru ke dalam tabel. Namun sebelumnya saya perlu menginstall module body-parser terlebih dahulu, agar express dapat membaca request body yang masuk ke server. Untuk menginstall modul body-parser, saya akan menggunakan perintah:

npm install --save body-parser

body-parser adalah middleware untuk express, oleh karena itu mari kita dafarkan dulu ke instance app dalam index.js.

Selanjutnya saya akan tambahkan endpoint (route) POST /buku pada index.js.

Endpoint POST /buku menerima 3 buah parameter request body yaitu judul, sinopsis, dan penulis. Setiap parameter request body bisa kita akses melalui object req.body .

Selanjutnya, untuk memasukan data baru ke dalam tabel bisa menggunakan method insert() . Method insert() harus diberikan argument object yang berisikan pasangan field dan data yang akan dimasukan ke dalam table.

Return value dari method insert() adalah field id dari data yang baru saja dimasukan. Type dari return value ini adalah array, jadi saya hanya akan mengambil data dari index ke 0 sebagai response.

Next,kita coba test endpoint POST /buku dengan menggunakan Postman.

Next, mari kita buat endpoint PUT /buku/:id untuk melakukan update data buku.

Pada baris ke-3 saya mengambil parameter id buku dari URL. Kemudian pada baris ke-8, saya melakukan update data buku dengan method where() dan diikuti dengan method update() . Method where() kita gunakan menuliskan klausa where dalam sql. Di sini saya ingin melakukan update data buku berdasarkan field id. Method update() menerima argument object, pasangan field dan data yang ingin diupdate.

Mari kita test kembali dengan Postman. Saya akan coba melakukan update data buku dengan id = 6.

Nice, data buku dengan id 6 berhasil diupdate!.

Terakhir mari kita buat endpoint DELETE /buku/:id untuk menghapus data buku

Untuk menghapus data buku, kita bisa gunakan method del() atau delete() . Pada baris ke-3 saya mengambil parameter id buku dari URL, kemudian pada baris-5 mengeksekusi method del() untuk menghapus data berdasarkan field id buku.

Oke mari kita test dengan menggunakan Postman. Saya akan coba menghapus buku dengan id = 6.
