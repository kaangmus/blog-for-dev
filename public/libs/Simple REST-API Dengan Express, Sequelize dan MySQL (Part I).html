Simple REST-API Dengan Express, Sequelize dan MySQL (Part I)

Buda Suyasa
May 6, 2018 · 5 min read

Express merupakan salah satu framework NodeJS yang sangat populer di kalangan web developer. Express sudah banyak digunakan oleh berbagai website besar seperti paypal.com, forbes.com, dan flickr.com.

Express sangat powefull namun sangat mudah dipelajari dan digunakan. Saat ini, Express banyak digunakan untuk membuat REST-API untuk single-page web application atau mobile application.

Dalam artikel ini saya akan menjelaskan bagaimana cara membuat REST-API sederhana yang terdiri dari operasi create, retrieve, update dan delete (CRUD) menggunakan Express, Sequelize dan database MySQL.

Untuk mempermudah interaksi dengan MySQL, kita akan menggunakan Sequelize. Sequelize adalah ORM database untuk mempermudah pemetaan database entry menjadi object, sehingga akan mempercepat dan mempermudah proses interaksi dengan database.

Kita akan langsung membuat REST-API dengan menggunakan Express, Sequelize dan MySQL. Saya asumsikan Anda sudah mengetahui dasar-dasar Javascript, MySQL dan konsep REST-API. Kita akan menempatkan seluruh kode yang ada dalam tutorial ini dalam 1 file agar lebih sederhana dan mudah dipahami. Anda bisa mendapatkan code akhir dari artikel ini di sini:

budasuyasa/book-rest-api

book-rest-api - Simple single file REST-API using ExpressJS, Sequelize, and MySQL

github.com

Untuk mempermudah testing, download Postman collection dari REST-API yang kita buat dalam tutorial ini di sini:
https://www.getpostman.com/collections/d07bc76008eb2d618c6f

Pertama pastikan Anda telah menginstall NodeJS pada komputer yang anda gunakan. Siapkan text editor favorit Anda dan buka Terimal (Linux/OSX) atau Comand Prompt (Windows). Saya menyarakan Anda menggunakan Visual Studio Code dengan plugin Terminal untuk mempermudah dalam proses coding.

Pertama, mari kita buat sebuah direktori baru dan buat sebuah Node package baru.

mkdir simple-rest && cd simple-rest && npm init

Perintah di atas berfungsi untuk membuat direktori dengan nama simple-rest dan untuk mebuat Node package baru. Anda akan diminta untuk meberikan informasi terkait package yang anda buat.

package name: (simple-rest)
version: (1.0.0)
description:
entry point: (index.js)
test command:
git repository:
keywords:
author:

Anda bisa mengisikan informasi package yang Anda inginkan seperti nama package, versi, deskripsi, entry point, dan lainnya. Jika sudah selesai, Node akan membuatkan file package.jsondi dalam direktori simple-rest yang berisikan informasi yang barusan Anda masukan.

Selanjutnya kita perlu menginstall beberapa package yang kita perlukan. Sebagai permulaan kita cukup hanya menginstall Express dan Sequelize saja. Ketikan perintah berikut pada Terminal/Command Prompt secara berurutan.

npm install --save express
npm install --save sequelize
npm install --save mysql2

Jika sudah selesai, kita akan coba untuk membuat program Hello World sederhana dengan Express untuk memastikan semuanya berkerja normal. Buatlah sebuah file baru dengan namaindex.js masukan code di bawah ini.

Kembali ke jendela Terminal/Command Prompt. Kita akan mencoba untuk menjalankan aplikasi yang kita buat. Ketikan perintah berikut untuk menjalankan server aplikasi Anda.

node index.js

Buka http://localhost:3000 menggunakan web browser. Jika tidak ada masalah, Anda akan menemukan tulisan Hello World pada browser Anda.

Pada tahap ini Anda sudah berhasil menginstall dan membuat sebuah aplikasi Hello World dengan Express. Mari kita lanjutkan dengan membuat sebuah database baru dengan menggunakan MySQL. Kita akan menamakan basis data ini simple-rest . Dalam basis data kita akan membuat sebuah tabel dengan nama book. Dalam REST-API yang kita buat ini, kita akan membuat beberapa endpoint untuk melakukan manipulasi CRUD dalam tabel book ini.

Sebelum lanjut lebih jauh, di dalam root directory project, mari kita tambahkan sebuah directory baru dengan nama public. Di dalam directory ini, kita tambahkan juga sebuah directory baru dengan nama img. Directory ini kita akan gunakan untuk menyimpan gambar buku yang nantinya bisa dipanggil dengan menggunakan URL. Berikut merupakan struktur directory project yang kita sedang buat.

|__index.js
|__node_modules
|__package.json
|__public
    └── img

Kini database dan struktur directory project kita sudah siap. Pertama kita akan melakukan konfigurasi koneksi database dengan Sequelize. Silahkan sesuaikan username dan password MySQL database Anda.

Langkah selanjutnya kita akan membuat sebuah model untuk merepresentasikan tabel book menjadi object. Melalui model inilah kita kita akan berinteraksi dengan basis data yang kita punya. Mari kita namai model yang kita buat dengan nama book.

Pada code di atas, Anda bisa melihat bahwa kita memetakan seluruh field yang ada dalam tabel book ke dalam sebuah object. Di sana kita mendefinisikan tipe data sesuai dengan tipe data yang didukung oleh Sequelize. Untuk field image kita mendefinisikan custom getter, di mana akan memberikan concat string /img/ di depan data field image. Kita menggunakan parameter freezeTableName dengan nilai true, untuk memberitahu Sequelize agar tetap menggunakan nama book saat melakukan query. Jika tidak, Sequelize akan menggunakan nama tabel books (plural), yang mana tidak tersedia dalam database kita.

Selanjutnya kita akan menentukan beberapa endpoint (routes) yang kita perlukan untuk operasi CRUD dalam REST-API yang kita buat. Berikut beberapa routes yang akan kita buat beserta fungsinya.

GET /book : route untuk mendapatkan semua data buku

POST /book : route untuk menambahkan data buku baru

PUT /book : route untuk mengedit/mengupdate data buku

DELETE /book/:isbn : route untuk menghapus data buku berdasarkan ISBN

Mari kita buat route /book untuk mendapakan semua data buku dalam tabel book. Tambahkan code berikut ke index.js.

Anda dapat simak dalam code di atas, apabila terdapat HTTP request GET ke path /book pada server REST-API kita, server akan merespon dengan memberikan semua data yang ada dalam tabel book. Dalam hal ini, kita menggunakan method findAll() pada model book untuk mengambil semua data pada tabel book.
